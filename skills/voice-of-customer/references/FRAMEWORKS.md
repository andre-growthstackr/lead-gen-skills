# VoC Frameworks Reference

Load this during Steps 5–6 (mining + synthesis).

## Pain / Value / Objection Taxonomy

### Pain sub-categories

Tag every pain quote with one of these to make clustering deterministic:

| Sub-category | Definition | Example phrasing |
|---|---|---|
| **Time tax** | Hours spent on manual or repetitive work | "Spend 8h/week pulling reports manually" |
| **Cost burden** | Direct or indirect $ cost of current state | "Paying 3 separate tools to do what one should" |
| **Data fragmentation** | Information scattered across systems | "Have to look in 4 places to answer one question" |
| **Quality gap** | Output is wrong, late, or unreliable | "Numbers in our weekly never match the CFO's" |
| **Capacity ceiling** | Can't grow without breaking | "Works for 50 leads/week but breaks at 200" |
| **Risk / compliance** | Exposure that keeps them up at night | "GDPR review keeps flagging this workflow" |
| **Emotional load** | Stress, embarrassment, anxiety | "Dreading the board meeting because of these numbers" |
| **Status / career** | Personal stakes — promotion, credibility | "I look like I don't know what I'm doing" |

### Value sub-categories

| Sub-category | Definition | Example phrasing |
|---|---|---|
| **Speed** | Faster than alternative | "Got results in 2 hours, would've been 2 weeks" |
| **Quality / accuracy** | More reliable than alternative | "Finally numbers I can trust without re-checking" |
| **Consolidation** | One thing replaces many | "Killed three tools and a Zapier subscription" |
| **Empowerment** | Person now does what required a specialist | "I can answer my own data questions now" |
| **Visibility / control** | Sees / controls something previously opaque | "I know exactly where every lead is" |
| **Confidence** | Reduced anxiety, increased trust | "Stopped second-guessing every email send" |
| **Recognition** | Got noticed / promoted because of it | "Got the credit for the new dashboard" |

### Objection sub-categories

| Sub-category | Definition | Resolution pattern |
|---|---|---|
| **Price** | "Too expensive" or "pricing unclear" | Show concrete ROI in customer's own units |
| **Switching cost** | "Migration scares us" | Show migration path + customer who did it |
| **Trust / brand** | "Never heard of you" or "you're new" | Case studies + recognizable logos |
| **Capacity / fit** | "Not sure it works for my size" | Segment-specific case + ICP confirmation |
| **Integration** | "Doesn't talk to my X" | Integrations page + Zapier fallback |
| **Internal politics** | "My boss / CFO would never approve" | Decision-maker-specific ROI calculator |
| **Risk aversion** | "What if it breaks our process?" | Free trial + reversibility guarantee |

### Trigger sub-categories

| Sub-category | Definition | Window to purchase |
|---|---|---|
| **Hiring trigger** | New role, replacement, headcount change | 2–8 weeks |
| **Funding trigger** | Round raised, runway extended | 4–12 weeks |
| **Growth pain** | Outgrew current tool or process | 1–4 weeks (urgent) |
| **Compliance trigger** | Regulation, audit, breach | 1–4 weeks (urgent) |
| **Competitive pressure** | Lost a deal because of capability gap | 4–8 weeks |
| **Tool failure** | Current tool broke / vendor shut down | 1–2 weeks (very urgent) |
| **Seasonal / planning cycle** | Q1 budget, end-of-fiscal-year | Predictable cadence |

## Jobs-to-be-Done (JTBD) Layering

For each pain you cluster, identify the JTBD job hidden inside:

> "When [situation], I want to [motivation], so I can [expected outcome]."

Example:
> "When my CFO asks for last month's revenue by source on Thursday, I want to
> hand her a report I trust, so I can spend Friday on strategy instead of
> reconciliation."

A pain cluster with no clear JTBD job is usually a *symptom* — keep mining
until you find the underlying job.

## Volume Thresholds

| Status | Required volume |
|---|---|
| **Anecdote** | < 10 quotes — useful for color, not for conclusions |
| **Indicator** | 10–50 quotes — directional, do not bet copy on it alone |
| **Pattern** | 50–100 quotes — strong enough for landing-page copy |
| **Conclusion** | 100+ quotes across ≥3 sources — strong enough for pricing or positioning decisions |

Aim for **Conclusion-level** evidence on top 3 pains and top 1 value driver.
Indicator-level is acceptable for trigger and objection clusters if time-boxed.

## Quote-Quality Checklist

Before including a quote in the report, verify:

- [ ] Verbatim — not paraphrased
- [ ] Source URL + date present
- [ ] Reviewer role / context attached if available (without revealing PII)
- [ ] Quote is ≥1 full sentence (skip one-word reviews unless intensity is extreme)
- [ ] Quote is not a marketing pull-quote from the company being reviewed
  (skip "world's best CRM" — those are useless)

**Strong quote signal:** specific number, specific name of competitor, specific
workflow step, specific timeframe. Vague language ("it's just better") is
filler — keep mining.

## Common Mistakes

- **Cherry-picking confirmation quotes.** If 4-star and 5-star reviews all say
  the same thing, you're confirming a marketing message, not researching VoC.
  Spend half your time in 1–3 star territory.
- **Conflating B2B and B2C language.** A B2B buyer says "ROI on data hygiene
  workflows." A B2C buyer says "this just feels easier." Same value, totally
  different copy.
- **Treating Reddit as gospel.** Reddit voices the *most engaged* fraction —
  high signal but skewed critical. Always pair with at least one paid-product
  review platform.
- **Paraphrasing into marketing speak.** "Streamlined operations" is *your*
  language. "I stopped having Sunday-night dread about Monday's report" is
  *theirs*. Use theirs.
